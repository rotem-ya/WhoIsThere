# הנחיות ל-cowork: הגשת עדכון גרסה 1.4.3

**מקור:** ריפו `rotem-ya/WhoIsThere`, ענף `claude/whoishere-visual-sound-rjcdzb`.
גרסה `1.4.3+103`. כל הטקסטים והמפרטים בענף.

**רקע:** 1.4.1 (עיצוב Candy) כבר חיה בגוגל. 1.4.3 = עדכון תוכן + פוליש מעליה: גלגל מזל
יומי, משימות יומיות, טבלת מובילים שבועית עולמית, ערכות רקע, מטבעות עפים, אנימציות
ומעברים חדשים. **אין שינוי בכללי המשחק.** חוקי ה-Firestore לטבלה השבועית **כבר נפרסו
וחיים** (2026-07-21) — אין פעולה נדרשת שם.

## 1) בילד אנדרואיד (AAB) → Google Play
- ⚠️ הפעלת בילדים דורשת אדם (אין הרשאת Actions ל-cowork). מבקשים מרותם להריץ:
  Actions → **Build AAB** → Run workflow → `build_name=1.4.3`.
- versionName יהיה 1.4.3; versionCode נגזר ממספר ריצת ה-workflow (עולה מעצמו). אם Play
  דוחה "version code already used" — להריץ שוב.
- Play Console → Production (או Testing לגל הדרגתי) → Create new release → העלה את
  ה-AAB → הדבק Release notes → הגש.

## 2) בילד iOS → TestFlight → App Store
- מרותם: לדחוף תג `ios-v*` מהענף → Codemagic בונה ומעלה ל-TestFlight אוטומטית.
- App Store Connect → גרסה חדשה **1.4.3** → קשר את הבילד מ-TestFlight → העלה צילומי
  מסך + Release notes → Submit for Review.

## 3) טקסטים
הכל מוכן ב-`docs/STORE_LISTING_TEXTS_v1.4.0.md` (תיאור קצר/מלא, כותרת משנה, טקסט קידום,
מילות מפתח, "מה חדש"). להעתיק כל שדה משם. **לא לשנות את שם האפליקציה.**
