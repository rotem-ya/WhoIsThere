# מרכז פרסים — עיצוב מלא (משחק + אדמין)

מסמך אמת יחיד לפיצ'ר "מרכז הפרסים": מאחד את כל דרכי קבלת המטבעות למקום אחד, מוסיף
רשימת מטלות יומיות/שבועיות, ומאפשר לאדמין לנהל את הכל בזמן אמת (גלגל, happy hour,
מטלות, ואירועים עתידיים) בלי עדכון גרסה.

**החלטות עיצוב (ננעלו):**
1. **מרכז פרסים עם טאבים** — חלון גלגל המזל הופך ל"מרכז פרסים" עם 3 טאבים:
   🎡 גלגל · 🎁 פרס יומי · ✅ מטלות. כניסה אחת (כפתור המתנות/גלגל בבית).
2. **Happy Hour מכפיל את כל רווחי המטבעות** (זכיות + פרס יומי + גלגל) בחלון הזמן.
3. **מטלות יומיות + שבועיות** כרשימה, כל אחת עם התקדמות ופרס משלה.
4. **הכל מנוהל מהאדמין** דרך מסמך Firestore אחד: `rewards_config/config_v1`
   (קריאה ציבורית, כתיבה isAdmin — בדיוק כמו `cosmetics_catalog/catalog_v1`).
5. **אקסטנסביליות** — הסכימה כוללת מערך `events` לאירועים/משחקים עתידיים; happy hour
   הוא סוג האירוע הראשון. מנגנון מטלות גנרי מאפשר להוסיף סוגי מטלות בלי בילד.

---

## חוזה Firestore — `rewards_config/config_v1`
קריאה ציבורית, כתיבה isAdmin. הקליינט ממזג עם ברירות מחדל מוטמעות (fallback), כך שאם
המסמך חסר או שדה חסר — המשחק מתנהג כמו היום. **כל הזמנים ב-UTC (ISO-8601).**

```json
{
  "version": 1,
  "spin": {
    "enabled": true,
    "segments": [10, 25, 50, 15, 100, 30, 75, 20],
    "weights":  [26, 18,  8, 22,   2, 14,  4, 20]
  },
  "happyHour": {
    "enabled": false,
    "multiplier": 2,
    "label": "שעת המזל!",
    "startUtc": "2026-07-25T17:00:00Z",
    "endUtc":   "2026-07-25T19:00:00Z"
  },
  "dailyQuests": [
    {"id": "win2",     "kind": "win",      "emoji": "🏆", "title": "נצחו 2 משחקים",      "target": 2, "reward": 40,  "active": true},
    {"id": "play3",    "kind": "play",     "emoji": "🎮", "title": "שחקו 3 משחקים",      "target": 3, "reward": 30,  "active": true},
    {"id": "discover5","kind": "discover", "emoji": "🗺️", "title": "גלו 5 מקומות חדשים", "target": 5, "reward": 50,  "active": true}
  ],
  "weeklyQuests": [
    {"id": "win15",     "kind": "win",      "emoji": "🏅", "title": "נצחו 15 משחקים השבוע", "target": 15, "reward": 150, "active": true},
    {"id": "discover20","kind": "discover", "emoji": "🌍", "title": "גלו 20 מקומות השבוע",  "target": 20, "reward": 120, "active": true}
  ],
  "events": []
}
```

### שדות ומגבלות
- `spin.segments` ו-`spin.weights` — שני מערכים **באותו אורך** (8 מומלץ, 4–12 נתמך).
  `weights` = משקלים יחסיים (לא אחוזים; הסכום חופשי). המשחק מצייר לפי משקל.
- `happyHour.multiplier` — מספר שלם 2–5. `startUtc`/`endUtc` — חלון פעיל. אם `enabled=false`
  או שאנחנו מחוץ לחלון — כפולה 1 (רגיל).
- `kind` של מטלה: אחד מ-`win` / `play` / `discover` (בעתיד `friends_win`). המשחק מודד
  התקדמות כדלתא של מונה מצטבר (בלי hooks פר-אירוע).
- `id` של מטלה — ייחודי ויציב (משמש למפתח ההתקדמות/claim פר-משתמש). **לא לשנות id
  של מטלה קיימת** — שינוי id = מטלה חדשה (מתאפסת).
- `active:false` מסתיר מטלה בלי למחוק אותה.
- `reward` — מטבעות (מוכפל ב-happy hour כמו כל רווח).

## צד המשחק (מה קלוד-המשחק בונה)
1. **`RewardsConfigService`** — קורא/מקאשר את `rewards_config/config_v1` (loadCached +
   snapshot חי), ממזג עם ברירות מחדל מוטמעות. חושף: `spinSegments`, `spinWeights`,
   `happyHourMultiplier(DateTime nowUtc)`, `dailyQuests`, `weeklyQuests`. סונכרן ב-`main`
   פעם אחת (כמו CosmeticsCatalogService).
2. **גלגל המזל** קורא segments/weights מהשירות (במקום מ-EconomyConfig הקבוע).
3. **Happy Hour** — `EconomyService` מכפיל את המטבעות בכל שלושת המסלולים (זכייה/פרס
   יומי/גלגל) ב-`happyHourMultiplier(now)`. תג "שעת המזל ×2" מוצג במרכז הפרסים ובבית.
4. **מטלות** — מנגנון רב-מטלות: לכל מטלה פעילה baseline+claimed פר-משתמש. יומי מתאפס
   ב-UTC day, שבועי ב-ISO week. State: `users/{uid}/economy/quests` (מכוסה בכלל
   ה-self-access הקיים; אין כלל Firestore חדש).
5. **מרכז פרסים (UI)** — חלון עם 3 טאבים; הטאב "מטלות" מציג את רשימת היומי+שבועי עם
   התקדמות וכפתור "אסוף". כרטיס המשימה היומית הבודד יוסר מהבית (עבר להאב) — "עושים סדר".

**עיקרון בטיחות:** בלי המסמך באדמין, השירות מחזיר ברירות מחדל מוטמעות → המשחק מתנהג
בדיוק כמו היום (אפס רגרסיה). happy hour כבוי כברירת מחדל.

---

## צד האדמין (מה קלוד-אדמין בונה) — ראה `docs/ADMIN_PROMPT_REWARDS_HUB.md`
מסך אדמין "🎁 מרכז פרסים" שקורא/כותב את `rewards_config/config_v1`:
- עורך פרוסות הגלגל (סכומים+משקלים) עם תצוגה מקדימה.
- מתג + עורך Happy Hour (כפולה, חלון זמן, תווית).
- CRUD למטלות יומיות/שבועיות (id/kind/emoji/title/target/reward/active).
- (עתידי) עורך `events`.

---

## תוכנית שחרור
1. אדמין בונה את מסך הניהול ויוצר את `rewards_config/config_v1` (ראה prompt).
2. משחק בונה את `RewardsConfigService` + happy hour + מרכז הפרסים.
3. **אין כלל Firestore חדש נדרש** (rewards_config קריאה ציבורית; אם צריך כלל כתיבה
   isAdmin — נוסיף ל-firestore.rules ונפרוס מ-main, כמו cosmetics_catalog).
4. בילד + פרסום. עד אז המשחק עובד עם ברירות המחדל.
