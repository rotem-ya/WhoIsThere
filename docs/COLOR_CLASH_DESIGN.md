# קרב צבעים (Color Clash) — מסמך תכנון משחק

> סטטוס: **טיוטה לאישור** — אין עדיין קוד. נכתב בעקבות רעיון המשתמש למשחק *אקטיבי*
> (לא רק צפייה בחשיפת משבצות).

## 1. החזון
היום השחקן **צופה** במשבצות נחשפות. כאן השחקן **פועל**: הדרך היחידה לחשוף את
התמונה היא לענות נכון על שאלות. שתי קבוצות-צבע מתחרות על אותו לוח; כל קבוצה
חושפת רק את המשבצות שלה. מי שמזהה את התמונה ראשון — מנצח.

## 2. הלולאה המרכזית
1. נבחרת תמונה מהקטלוג הקיים (`RoomService._loadLocalImages(categoryId)`).
2. כל שחקן משויך לצבע: **כחול** או **אדום** (סולו = כחול מול בוט אדום).
3. מופיעה **שאלת רב-ברירה** (4 תשובות).
4. תשובה נכונה → נחשפת **משבצת אחת** בצבע השחקן.
5. **ראייה א-סימטרית:** כל שחקן רואה רק את המשבצות שצבעו כבש. היריב יכול כבר
   לראות חצי תמונה בעוד אתה בהתחלה — מתח.
6. כשמזהים → כפתור "נחש!" + הקלדה. ניחוש נכון = ניצחון הסבב.

## 3. הכרעות תכנון (ברירות מחדל מומלצות — לאישור)
| נושא | ברירת מחדל מומלצת | חלופה |
|------|--------------------|--------|
| ראייה | **א-סימטרית** (רק הצבע שלך) | לוח משותף |
| מקור שאלות | **מאגר טריוויה כללי** (רב-ברירה) | שאלות פר-תמונה |
| פורמט תשובה | **רב-ברירה (4)** | טקסט חופשי |
| תנאי ניצחון | **ראשון שמנחש את התמונה** | רוב משבצות |
| הרכב | **MVP: סולו מול בוט**, אז 1v1, אז קבוצות | — |

**למה טריוויה כללית:** אותו מאגר עובד לכל תמונה → מנתק את צוואר-בקבוק התוכן
(לא צריך שאלה ייעודית לכל תמונה). האדמין מוסיף שאלות כמו שמוסיף מקומות.

## 4. State חדש בחדר (RoomModel)
- `mode: 'colorClash'` — מבדיל מהמשחק הרגיל/היט.
- `teamOf: Map<playerId, 'blue'|'red'>` — שיוך צבע.
- `tileOwner: Map<tileIndex, 'blue'|'red'>` — מי כבש כל משבצת (נחשף).
- `currentQuestionId` / מצב שאלה נוכחית פר-שחקן (או תור).
- תנאי ניצחון: `winnerId` קיים — נשען על הקיים.
- חשיפה פר-צופה: ה-UI מציג רק `tileOwner[i] == myTeam` (לא משנה את מבנה
  ה-`placedPieces` המשותף — נפרד).

## 5. תוכן חדש — מאגר שאלות
- אוסף Firestore חדש: `trivia_questions` (או במניפסט) — חוזה אדמין מקביל
  ל-`content_manifest`. כל שאלה: `{ q, answers[4], correctIndex, difficulty?,
  topic? }`.
- מאגר מוטמע (bundled) כרשת ביטחון + הרחבה מהאדמין (כמו התמונות).
- **לתעד בחוזה המשותף** `docs/CONTENT_PLATFORM.md` (סעיף חדש + Changelog,
  בשני הריפו) — קלוד האדמין יבנה CRUD לשאלות.

## 6. UI
- מסך משחק חדש/וריאנט: לוח עם משבצות צבועות (כחול/אדום) — רק שלך גלויות.
- כרטיס שאלה (4 כפתורי תשובה גדולים, נוח למובייל) + טיימר קצר.
- אינדיקטור התקדמות יריב ("היריב חשף 6/?") — מתח בלי לחשוף לו את התמונה.
- כפתור "נחש!" קבוע (כמו היום).
- סאונדים קיימים: תשובה נכונה = `correct_ding`, שגויה = `wrong_buzz`,
  חשיפה = `aperture_open`, ניצחון = `victory_fanfare`.

## 7. בוטים
- בוט אדום עונה נכון בהסתברות לפי קושי + קצב; חושף משבצות אדומות בקצב,
  ומנחש כשחשף מספיק (כמו לוגיקת הבוט הקיימת ב-`game_board_screen`).

## 8. כלכלה
- רוכב על הקיים: דמי כניסה/פרס לפי מצב (כמו משחק רגיל מול חברים).
- קוסמטיקה (אווטרים/מסגרות/שמות/אפקטים) עובדת אוטומטית — אותו `PlayerModel`.

## 9. שלבי מימוש (מוצע)
1. **מאגר שאלות** — model + קטלוג מוטמע + שירות טעינה (`TriviaService`).
2. **State** — הרחבת `RoomModel` (mode/teamOf/tileOwner) + יצירת חדר colorClash.
3. **לולאת סולו** — מסך שאלה + חשיפה פר-צבע + ניחוש + בוט. (MVP מלא בסולו)
4. **מולטיפלייר** — סנכרון `tileOwner`/שאלות דרך Firestore, 1v1.
5. **קבוצות** — יותר משחקן לצבע.
6. **חוזה אדמין** — תיעוד + פרומפט לקלוד האדמין לניהול שאלות.

## 10. סיכונים
- **ראייה א-סימטרית** דורשת הקפדה: לעולם לא לשלוח/להציג לצופה משבצות של היריב.
- **מאגר שאלות בעברית** — צריך תוכן איכותי (התחלה: ~100 שאלות מוטמעות).
- בדיקה חיה חיונית (אנימציות חשיפה, סנכרון, בוט).
