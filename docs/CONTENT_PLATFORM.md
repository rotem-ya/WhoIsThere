# פלטפורמת תוכן WhoIsThere — חוזה משותף (משחק ⇄ אדמין)

> **מסמך זה הוא מקור האמת היחיד** המשותף לשני הפרויקטים:
> - **`rotem-ya/whoisthere`** — אפליקציית המשחק (Flutter). מתחזק: "קלוד המשחק".
> - **`rotem-ya/Guess_The_Place_Admin`** — אפליקציית האדמין. מתחזק: "קלוד האדמין".
>
> כל שינוי בחוזה (סכמה, שמות שדות, קטגוריות, נתיבי Storage, rules) **חייב** להתעדכן כאן **בשני הריפו במקביל** ולהירשם ב-[Changelog](#changelog--סנכרון-פיצרים) בתחתית. עותק זהה של הקובץ נשמר בשני הריפו (`docs/CONTENT_PLATFORM.md`). אם מצאת אי-התאמה בין הקוד למסמך — המסמך מנצח; תקן את הקוד או עדכן את המסמך ותעד.

## 1. עיקרון העל — פלטפורמה אחת, שני משחקים, עתיד פתוח

יש כיום **שני סוגי משחק** הרוכבים על אותה תשתית תוכן:

| משחק | קטגוריות (`category`) | רשת משחק |
|------|------------------------|-----------|
| **זיהוי מקומות** (הרגיל) | `israel_places` | 6×6 |
| **חי צומח דומם** (מקצה מהיר, 3 סבבים) | `animals` (חי) · `plants` (צומח) · `objects` (דומם) | 10×10, קלף/שנייה |
| (עתיד) | `world_sites` · `israel_figures` · `world_figures` | — |

**הפרדה קריטית — שתי אפליקציות נפרדות בכוונה:**
- אפליקציית **המשחק** מכילה את כל לוגיקת המשחק. היא **קוראת** תוכן מהענן.
- אפליקציית **האדמין** מכילה **רק** ניהול תוכן (CRUD על המניפסט + העלאת תמונות). **אסור** שתכיל לוגיקת משחק.
- שתיהן חולקות **רק**: מסמך המניפסט ב-Firestore, ה-Storage, וקובץ זה.

**כלל זהב להוספת משחק עתידי:** מוסיפים קטגוריה חדשה (id + JSON בצד המשחק). שום פיצ'ר תוכן לא מקודד לפי סוג משחק — הכל עובר דרך `category`. כך משחק חדש מקבל ניהול-אדמין בחינם.

## 2. מודל הנתונים — מסמך מניפסט יחיד ב-Firestore

**נתיב:** `content_manifest/places_v1` (מסמך בודד; קריאה אחת בכל הפעלת אפליקציה).

```jsonc
// content_manifest/places_v1
{
  "version": 7,            // int — bump בכל פרסום. המשחק משתמש לטלמטריה/דיבוג.
  "places": [              // מערך של כל המקומות שהאדמין מנהל
    {
      "id": "western_wall",        // string יציב — מפתח. חובה, ייחודי, לא ריק.
      "isActive": true,            // bool — ברירת מחדל true. false = מוסתר מהמשחק.
      "source": "bundled",         // "bundled" | "remote". ברירת מחדל "bundled".
      "category": "israel_places", // אחד מה-ids בטבלה למעלה. ברירת מחדל "israel_places".
      "nameHe": "הכותל המערבי",     // (גם name_he מתקבל)
      "answerHe": "הכותל",          // (גם answer_he). נופל ל-nameHe אם חסר.
      "aliasesHe": ["הכותל המערבי", "הכותל"], // (גם aliases_he). תשובות חלופיות מקובלות.
      "facts": ["...", "..."],     // רמזים. רלוונטי ל-remote בלבד (ראה §4).
      "imageUrl": "https://.../western_wall.jpg", // remote בלבד — download URL מ-Storage.
      "imageVersion": 3            // remote בלבד — int. bump בכל החלפת תמונה (cache-bust).
    }
  ]
}
```

**חוקי פרסום (publish):**
1. כתוב את **כל** המסמך בכתיבה אחת (set), כולל `version` מוגדל.
2. אל תסיר שדות לא-מוכרים שכבר קיימים — שמר תאימות קדימה.
3. שמות שדות מקובלים בשני וריאנטים (camelCase מועדף; snake_case נתמך): `nameHe`/`name_he`, `answerHe`/`answer_he`, `aliasesHe`/`aliases_he`.

---

## 3. אחסון תמונות (Firebase Storage)

- מקומות **remote** בלבד זקוקים לתמונה ב-Storage. מקומות **bundled** — התמונה כבר באפליקציה.
- נתיב מומלץ: `place_images/{category}/{id}.jpg` (או `.webp`). שמור על דחיסה (JPEG/WebP, ~150KB) — ה-egress עולה כסף.
- האדמין מעלה את הקובץ → משיג `getDownloadURL()` → כותב אותו ל-`imageUrl` בכניסת המקום במניפסט.
- **החלפת תמונה לאותו `id`:** העלה מחדש **והגדל את `imageVersion`**. בלי זה — המכשירים ימשיכו להציג את התמונה הישנה מהמטמון.

## 4. סמנטיקה — bundled מול remote (קריטי להבנת מה האדמין שולט בו)

| | **bundled** | **remote** |
|---|---|---|
| מקור התמונה | מוטמע באפליקציה (assets) | Firebase Storage |
| מקור המטא-דאטה (שם/רמזים) | JSON מוטמע באפליקציה | **המניפסט** (`nameHe`/`facts`/…) |
| מה האדמין שולט בו | **`isActive` בלבד** (הפעלה/השבתה) | **הכל** (הוספה/עריכה/השבתה/החלפת תמונה/מחיקה) |
| נכנס לבריכת המשחק כש… | קיים ב-JSON **וגם** `isActive!=false` | `isActive==true` **וגם** התמונה כבר במטמון המכשיר |

- מקום **bundled** שלא מופיע במניפסט נחשב **פעיל** (ברירת מחדל). כדי להשבית אותו — הוסף לו כניסה עם `isActive:false`.
- כדי שהאדמין יציג **את כל** המקומות ה-bundled (כדי לאפשר השבתה), הוא צריך את רשימת ה-ids המוטמעים. ראה **[`docs/content_catalog_seed.json`](content_catalog_seed.json)** — snapshot מלא (id, category, nameHe) לייבוא חד-פעמי. יש לרענן אותו כשתוכן מוטמע משתנה (ראה Changelog).
- **השבתה אינה מוחקת היסטוריה:** השבתת מקום מורידה אותו מהבריכה לעתיד, אבל מקום שכבר גולה ע"י שחקן נשאר ב-`discoveredImageIds` שלו (היסטוריה אישית). זו התנהגות מכוונת.

## 5. מה אפליקציית האדמין חייבת לספק

1. **דשבורד בזמן אמת** — `snapshots()` על `content_manifest/places_v1` (לא `get()` חד-פעמי). מציג:
   - סה"כ מקומות פעילים / כלל המקומות, **מפולח לפי קטגוריה** (זיהוי מקומות / חי / צומח / דומם בנפרד).
   - לכל קטגוריה: רשימת המקומות עם מצב (פעיל/מושבת), source (מוטמע/ענן), ותמונה ל-remote.
   - **חשוב:** הספירה חייבת לכלול גם bundled (מ-seed) וגם remote, כדי שלא תיווצר אי-התאמה מול המשחק (באג עבר: אדמין הראה 39, אפליקציה 41).
2. **הפעלה/השבתה** — toggle על `isActive` לכל מקום (bundled או remote).
3. **הוספת מקום remote** — טופס: בחירת קטגוריה (חובה!) → העלאת תמונה ל-Storage → מילוי nameHe/answerHe/aliasesHe/facts → כתיבת כניסה עם `source:"remote"` + `imageUrl` + `imageVersion:1`.
4. **עריכת/החלפת** מקום remote — כולל החלפת תמונה עם bump ל-`imageVersion`.
5. **מחיקת** מקום remote (bundled לא נמחק, רק מושבת).
6. **פרסום** — כל שינוי כותב מחדש את המסמך עם `version` מוגדל (אפשר batching של כמה שינויים לפרסום אחד).

**ניהול לפי משחק (ההפרדה שהמשתמש ביקש):** ה-UI חייב להציג ולאפשר עריכה **בנפרד לכל קטגוריה/משחק** — טאב/מסנן ל"זיהוי מקומות" וטאב נפרד ל"חי / צומח / דומם". בחירת קטגוריה בהוספת מקום היא **חובה** — מקום בלי `category` נכנס כברירת מחדל ל-`israel_places` (המשחק הרגיל) בלבד ולא יופיע בחי צומח דומם.

## 6. הרשאות ואבטחה

- **Firestore rules** (בריפו המשחק, `firestore.rules`): קריאה ל-`content_manifest` — לכל מחובר; כתיבה — `isAdmin()` בלבד.
- **`isAdmin()`** = claim מותאם `admin==true` **או** `request.auth.token.email` ברשימת ההיתר. כיום הרשימה: `rot4735@gmail.com`. כל שינוי ברשימה — לעדכן ב-`firestore.rules` **וב-`storage.rules`** של ריפו המשחק (הם מקור האמת ל-rules).
- האדמין **חייב** להיות מחובר עם חשבון Google `rot4735@gmail.com` (לא אנונימי), והדומיין של האדמין חייב להופיע ב-Firebase Console → Authentication → Settings → Authorized domains.
- Storage rules: קריאה למחובר, כתיבה ל-`isAdmin()` בלבד.

## 7. נקודות החנק בצד המשחק (לידיעת קלוד האדמין — אל תיגע, רק כדי להבין לאן התוכן זורם)

- `ContentManifestService` (`lib/services/content_manifest_service.dart`): קורא את `content_manifest/places_v1` פעם אחת בהפעלה (`loadCached`+`sync` ב-`main.dart`), שומר offline, ומחמם מטמון לתמונות remote פעילות.
- `RoomService._loadLocalImages({categoryId})`: ממזג מוטמע + remote **לפי קטגוריה** → זו הבריכה לכל סוגי המשחק. חי צומח דומם קורא לה לכל קטגוריה דרך `_buildHeat`.
- ids הקטגוריות בקוד: `lib/core/constants/game_categories.dart` — **חייבים** להיות זהים למחרוזות `category` במניפסט.
