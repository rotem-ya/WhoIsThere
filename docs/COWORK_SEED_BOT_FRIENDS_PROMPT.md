# פרומפט ל-cowork — הרצת "Seed Bot Friends" ל-7 חברים-בוט

עבוד עם רותם **שלב-שלב**: הסבר מה ללחוץ, המתן לאישור/צילום מסך, המשך רק אחרי שהצעד הקודם הצליח בפועל. אין ל-cowork הרשאה להריץ GitHub Actions דרך ה-API — זו לחיצה אנושית בממשק.

## מה זה עושה
מריץ workflow חד-פעמי שיוצר ב-Firestore 7 "חברים" מדומים (בוטים) ומצרף אותם לרשימת החברים של **חשבון אחד בלבד** — לא נוגע באף משתמש אמיתי אחר. המטרה: לאפשר לרותם לבדוק את חוויית "משחק עם חברים" (רשימת חברים, קבוצות, טבלת ניקוד, הזמנות, ובדיקת פוש) בלי לגייס בודקים אמיתיים.

## שלב 1 — איזה אימייל?
שאל את רותם: **עם איזה אימייל הוא מחובר בפועל באפליקציית WhoIsThere** (Google/Apple sign-in) — זה לא בהכרח אותו אימייל שהוא משתמש בו כאן. חשוב לוודא שהוא כבר פתח את האפליקציה והתחבר לפחות פעם אחת (ה-workflow דורש ש-`users/{uid}` כבר קיים).

## שלב 2 — הרצת ה-workflow
1. עבור ל-GitHub → ריפו `rotem-ya/whoisthere` → טאב **Actions**.
2. בסיידבר השמאלי, בחר **Seed Bot Friends (dev/testing)**.
3. **Run workflow** (כפתור בצד ימין):
   - Branch: `claude/whothere-v111-launch-iqkbq2`
   - `target_email`: האימייל משלב 1
4. **Run workflow**. המתן שהריצה תסתיים (בערך דקה).

## שלב 3 — אימות
פתח את הריצה שהסתיימה, בדוק בלוג:
- ✅ הצלחה: `Resolved <email> -> <uid>` ואז 7 שורות `+ bot_friend_0X (<שם>) -> friend of <uid>` ואז `Done.`.
- ❌ כישלון: אם כתוב `there is no user record corresponding to the provided identifier` — האימייל לא נכון או שהמשתמש עדיין לא נרשם ב-Firebase Auth; חזור לשלב 1 עם אימייל אחר. אם כתוב `users/{uid} does not exist` — רותם צריך לפתוח את האפליקציה ולהתחבר קודם, ואז לנסות שוב.

## שלב 4 — מה רותם אמור לראות באפליקציה
אחרי הצלחה, בבילד העדכני ביותר (r5+):
- טאב **חברים** מציג 7 חברים חדשים (אריאל, נועה, עמית, שירה, דניאל, מאיה, ליאור).
- לחיצה על אייקון 🧪 ליד כל אחד מהם פותחת תפריט לבדיקת פוש (הזמנת משחק עכשיו/בעוד דקה, או הודעת צ׳אט).
- יצירת חדר חברים → בלובי מופיע כפתור "הוסף בוט (לבדיקה)" למילוי סלוטים ומשחק מלא לבד.
