# חוזה הפעלה/הסתרה של מקומות (Place Activation Contract)

> מסמך זה הוא ה"חוזה" בין **אפליקציית הניהול (אדמין)** לבין **המשחק הראשי**.
> הוא מתאר איך האדמין מפעיל/מסתיר מקומות — בודד או באצווה — בלי למחוק נתונים,
> בלי לשנות קוד, ובאופן הפיך לחלוטין.
>
> גישה זו עובדת על **נתונים מקומיים בלבד** (asset מצורף לאפליקציה). היא אינה
> תלויה בענן (Firestore/Storage). עדכון "חי" בזמן אמת הוא **שלב הבא מכוון**
> (ראו סעיף "עדכון חי" בסוף) ואינו מופעל כרגע.

---

## 1. מקור האמת: השדה `is_active`

כל מקום בקטלוג נושא שדה בוליאני `is_active`.

| ערך `is_active` | התנהגות במשחק |
|---|---|
| `true` | המקום **פעיל** — מוגרל ומוצג לשחקנים כרגיל |
| `false` | המקום **מוסתר** — נשאר בנתונים, אך **לא מוגרל ולא מוצג** |
| השדה חסר | נחשב **פעיל** (ברירת מחדל לתאימות לאחור — נתונים ישנים לא נשברים) |

- **הסתרה = שינוי נתונים בלבד** (`is_active: false`). אין שינוי קוד.
- **הפיך תמיד** — החזרה ל-`true` מחזירה את המקום למשחק.
- **בטוח** — המשחק מוגן מקריסה גם אם נשארים מעט מאוד מקומות פעילים (או אפס:
  המשחק פשוט לא מתחיל סבב, ללא קריסה). נבדק גם במצב **סולו/בוטים**.

מיקום הקובץ במשחק הראשי:
```
assets/game_places/data/israel_places.json
```

מימוש הכיבוד במשחק: `lib/services/room_service.dart` → `_readLocalImages()`
(`.where((place) => place['is_active'] != false)`). זהו מקור יחיד שכל נתיבי
המשחק עוברים דרכו (מולטיפלייר, quick-match, וגם סולו/בוטים).

## 2. מבנה רשומת מקום (לעיון)

```jsonc
{
  "id": "masada",                      // מזהה ייחודי — מפתח המיזוג
  "name_he": "מצדה",
  "answer_he": "מצדה",
  "category": "landmark",
  "city_he": "...",
  "image_asset": "assets/game_places/images/masada.jpg",
  "difficulty": "easy",
  "is_active": true,                   // ← השדה שהאדמין שולט בו
  "facts": [ "...", "..." ]
}
```

המיזוג נוגע **אך ורק** ב-`is_active`. כל שאר השדות נשמרים כפי שהם.

---

## 3. פורמט ה-Patch שהאדמין מייצא (תומך אצווה)

אפליקציית האדמין מייצאת קובץ JSON אחד עם **רשימת עדכונים** — מקום אחד או
רבים בבת אחת (Bulk):

```json
{
  "schema": "whoisthere.place-activation/v1",
  "generated_at": "2026-06-12T12:00:00Z",
  "updates": [
    { "id": "masada",       "is_active": false },
    { "id": "dead_sea",     "is_active": false },
    { "id": "western_wall", "is_active": true  }
  ]
}
```

כללים:
- `updates` הוא **מערך** → אצווה היא טבעית (כל מספר מזהים בבת אחת).
- כל פריט חייב: `id` (מחרוזת) + `is_active` (בוליאני).
- אפשר לכלול רק את המקומות שהשתנו — אין צורך לייצא את כל הקטלוג.
- `schema` ו-`generated_at` מומלצים אך לא חובה.

---

## 4. נקודת העדכון: מיזוג אטומי לפי `id`

הכלי שמיישם את ה-Patch על הקטלוג:

```
scripts/apply_place_activation.js
```

הרצה:
```bash
# תצוגה מקדימה ללא כתיבה
node scripts/apply_place_activation.js patch.json --dry-run

# החלה בפועל (על קובץ ברירת המחדל)
node scripts/apply_place_activation.js patch.json

# או על קובץ ספציפי
node scripts/apply_place_activation.js patch.json assets/game_places/data/israel_places.json
```

הכלי מבטיח את כל דרישות הבטיחות:
- **מיזוג לפי `id`** — לא החלפת הקובץ כולו. רק `is_active` של המזהים ב-Patch מתעדכן.
- **לעולם לא מוחק / לא מוסיף מקומות** — אם ספירת המקומות משתנה, הכלי מסרב לכתוב.
- **כתיבה אטומית** — כתיבה לקובץ זמני ואז `rename` (קורא לעולם לא רואה קובץ חצי-כתוב).
- **id לא מוכר** מדווח ומדולג — לא קורס.
- **הפיך** — `is_active: true` ב-Patch מחזיר מקום מוסתר.

פלט לדוגמה:
```
Patch applied to assets/game_places/data/israel_places.json:
  places: 41 (unchanged total — merge by id)
  activated: 0, hidden: 2, no-op: 1
  skipped 1 unknown id(s): does_not_exist
  written atomically.
```

## 5. הזרימה המלאה (כיום)

1. **אדמין**: לוחץ הפעלה/הסתרה (בודד או בחירה מרובה) → מייצא `patch.json` בפורמט שבסעיף 3.
2. **מיזוג**: מריצים `node scripts/apply_place_activation.js patch.json` → מעדכן את `israel_places.json` בבטחה.
3. **בנייה**: ה-asset המעודכן נכלל בבנייה הבאה → המשחק מכבד את המצב החדש.

> כיוון שהמצב נשמר ב-asset מצורף, השינוי משתקף במשחק ב**בנייה הבאה**.
> אם רוצים שהאדמין יעדכן ישירות את הקובץ במאגר, הוא יכול לייצא קטלוג מלא ממוזג,
> אך **מומלץ לעבור דרך כלי המיזוג** כדי להבטיח שמקומות לא יימחקו בטעות.

---

## 6. אזהרה: הוספת מקום **חדש לגמרי**

הסתרה/הפעלה של 41 המקומות הקיימים עובדת מיידית. אך מקום עם `id` שלא קיים היום
מסונן בשכבת הגנה שנייה: allowlist בשם `_availableLocalPlaceIds`
(ב-`room_service.dart`), שמוודאת שלכל מקום שמוצג יש קובץ תמונה תקין.

כדי להוסיף מקום **חדש** (לא רק להפעיל קיים), צריך גם:
1. להוסיף את רשומת המקום ל-`israel_places.json` (כולל `image_asset`).
2. להוסיף את קובץ התמונה תחת `assets/game_places/images/`.
3. להוסיף את ה-`id` ל-`_availableLocalPlaceIds`.

זהו צעד נפרד מהפעלה/הסתרה ואינו נכלל בחוזה הנוכחי.

---

## 7. עדכון "חי" (שלב הבא — לא פעיל כעת)

עדכון בזמן אמת (שינוי באדמין שמשתקף במשחק ללא בנייה מחדש) דורש מקור הניתן
לשינוי בזמן ריצה (קובץ קונפיג מרוחק / Firestore). זה **הוקפא בכוונה** כדי לא
להכניס סיכון לפני השקה. כשנרצה להוסיף זאת, החוזה לא ישתנה: אותו פורמט Patch
ואותה לוגיקת מיזוג-לפי-id יוחלו על overlay בזמן ריצה, מעל ה-asset הבסיסי.
