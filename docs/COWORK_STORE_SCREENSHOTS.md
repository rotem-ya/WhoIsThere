# הנחיות ל-cowork: הגשת עדכון "Candy" — בילד + צילומי מסך (Google Play + App Store)

עדכון ריברנד: האפליקציה עברה לקו עיצוב **Candy** (סגול-ענב, כרטיסי ג'לי, סקינים חדשים
מרונדרים בקוד, לוח גדול יותר, ראש מהודק, אנימציות פרימיום). **אין שינוי בכללי המשחק** —
עיצוב בלבד. המשימה: להעלות לשתי החנויות את **הבילד האחרון** ואת **צילומי המסך החדשים**,
בעברית (he-IL).

---

## 0. קדם-תנאים לבילד (חשוב)
- כל שינויי Candy + הפוליש נמצאים בענף **`claude/whoishere-visual-sound-rjcdzb`**.
- ⚠️ **גרסה 1.4.1 (Candy) כבר חיה בגוגל.** הבילד הבא הוא **1.4.3** — עדכון פוליש מעליה
  (מטבעות עפים, לובי, מעברים, חגיגת פרימיום). הענף כבר מסומן `1.4.3+103`.
- ⚠️ **versionCode חייב להיות גבוה מזה של 1.4.1 החי.** ב-Build AAB ה-versionCode נגזר
  ממספר ריצת ה-workflow (`github.run_number`) שרק עולה, אז ריצה חדשה בדרך כלל תספיק —
  אבל אם Play דוחה על "version code already used", להריץ שוב (המספר יעלה).
- לפי כללי הריפו, בונים **רק מענף ההשקה** `claude/qa-launch-prep-EXqLn`. לכן ראשית
  **למזג** את ענף Candy אל ענף ההשקה (או לבנות מענף Candy אם רותם מאשר).
- ⚠️ **הפעלת בילדים דורשת אדם** (ל-cowork/agent אין הרשאת Actions/tag) — לבצע דרך רותם.

## 1. בילד אנדרואיד (AAB) → Google Play
בונים AAB חדש לגרסה **1.4.3** (הפוליש מעל ה-Candy החי):
workflow **Build AAB** ב-Actions → `build_name=1.4.3` (רותם מריץ, ל-cowork אין הרשאת Actions).
ה-versionName יהיה **1.4.3** וה-versionCode ממספר ריצת ה-workflow (גבוה מ-1.4.1), חתום **EA:3B**.
1. Play Console → האפליקציה → **Production** (או Testing אם רוצים גל הדרגתי) →
   **Create new release** → העלה את ה-AAB שנבנה.
2. **Release notes (עברית):** ראה נוסח למטה. שמור והגש לבדיקה.

## 2. בילד iOS → TestFlight → App Store
1. דחוף תג **`ios-v*`** מענף ההשקה → Codemagic בונה IPA ומעלה ל-TestFlight אוטומטית.
2. App Store Connect → האפליקציה → צור/בחר גרסה חדשה (1.4.3) → קשר את הבילד מ-TestFlight.
3. העלה צילומי מסך (סעיף 4) + Release notes → **Submit for Review**.

---

## 3. צילומי מסך — הקבצים מוכנים
רותם יעביר `candy_store_screens.zip` + `candy_ipad_screens.zip`. הכל כבר ממוסגר עם
כותרות עברית וברקע Candy, במידות המדויקות של כל סלוט:

- **`apple/`** — 1290×2796 (iPhone 6.7") → App Store Connect, טאב iPhone.
- **`ipad/`** — 2048×2732 (iPad 13"/12.9") → App Store Connect, טאב iPad → iPad 13" Display.
- **`play/`** — 1080×2160 (יחס 2:1) → Google Play, צילומי טלפון.

⚠️ באפל יש **שני** מדורים נפרדים (iPhone + iPad) — להחליף את הישנים בשניהם.
אם ASC דורש דווקא 2064×2752 ל-13", לבקש מקלוד-המשחק גרסה בגודל הזה.

**סדר הקבצים (כבר ממוספר 01–07):**
1. `01_home` — בחרו סוג משחק
2. `02_blue` — חשפו את התמונה ונחשו ראשונים
3. `03_boardstore` — רקעים בכל צבע
4. `04_magen` — סקיני פרימיום נוצצים
5. `05_letters` — משחק האותיות
6. `06_heat` — חי-צומח-דומם
7. `07_win` — ניצחו ואספו מטבעות

## 4. העלאת הצילומים
- **Google Play:** Play Console → הופעה בחנות → דף החנות הראשי → צילומי מסך של טלפון →
  מחק ישנים, העלה את כל `play/` בסדר 01→07. שמור.
- **App Store:** App Store Connect → הגרסה → App Previews and Screenshots → גודל 6.7" →
  העלה את כל `apple/` בסדר 01→07. שמור.
